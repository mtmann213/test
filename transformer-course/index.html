Tokenization

How text becomes numbers — the first step every LLM takes

Models don't read text directly. They read integer IDs. The step that converts your prompt into a sequence of those integers is called tokenization.

A tokenizer takes a string and produces a sequence of integers, where each integer points to an entry in a fixed vocabulary. Modern LLM vocabularies usually contain tens of thousands to a few hundred thousand entries.

Tiny explainer: token ID
A token ID is the integer the model uses for one vocabulary entry. The model works with the number, not the written word itself.

Tokens aren't usually whole words — they're subword pieces. "Tokenization" might split into ["token", "ization"]. "Running" becomes ["run", "ning"]. This is efficient: common pieces become single tokens, rare or novel words get composed from smaller pieces.

The classic example: ask an LLM how many R's are in "strawberry." Older models got it wrong. That's not the model failing at counting — it's the model not operating on letters directly, only token IDs that happen to spell out a word a human would split letter by letter.

Embeddings

How numbers get meaning — the lookup table that gives tokens life

A token ID like 1024 is just a row index. It doesn't mean anything by itself. The thing that gives it meaning is a giant table called the embedding matrix.

Every model has one. It has one row per vocabulary entry, and each row is a long vector of numbers — typically 4,096 dimensions for a 7B-class model.

Tiny explainer: embedding matrix
The embedding matrix is a lookup table. Token ID in, learned vector out. Each row is the model's representation of what that token "means."

The interesting property: semantically similar tokens end up with similar vectors. "King" is close to "queen," "Paris" is close to "France." None of this is hard-coded — it emerges from training on enough text.

Positional Encoding

How the model knows what order tokens came in — without it, everything is a bag of words

The transformer's attention mechanism is permutation-invariant. If you shuffle the input tokens, the attention scores stay exactly the same. That means without extra information, "The cat chased the dog" and "The dog chased the cat" would look identical.

Positional encodings solve this by adding position-specific vectors to each token's embedding. The most common approach uses sine and cosine waves of different frequencies — one per dimension.

Tiny explainer: why sines and cosines?
Sine/cosine positional encodings let the model learn to reason about relative positions. The encoding for position N+1 is a fixed function of the encoding for position N, regardless of what N is. This generalizes to sequences longer than anything seen during training.

Attention

How tokens share information with each other — the core mechanism of transformers

Self-attention is how every token in a sequence looks at every other token and decides: "how much should I pay attention to you?"

The math works like this: each token gets three vectors — a Query (what am I looking for?), a Key (what do I contain?), and a Value (what information can I share?). The query of one token is compared against the keys of all tokens. High scores mean "pay attention."

Tiny explainer: Q, K, V analogy
Think of it like a database lookup. Your Query is what you're searching for. Keys are the labels on file cabinets. Values are the contents inside. You compare your query to each key — the best matches tell you which files to read.

The attention scores go through softmax (so they sum to 1), then weight the values. The result is a new representation for each token that incorporates information from all the tokens it attended to.

Multi-Head Attention

How the model tracks many kinds of relationships at once — parallel conversations between tokens

A single attention head can only learn one kind of relationship. But language has many: syntax, semantics, coreference, negation, agreement...

Multi-head attention runs multiple attention heads in parallel, each with its own Q/K/V projections. Each head learns to focus on different aspects:

  • One head might track subject-verb agreement
  • Another might link pronouns to their antecedents
  • Another might capture local word order (bigrams, trigrams)

The outputs of all heads are concatenated and linearly projected. The model learns which head matters for what.

Feed-Forward Network

Where a large share of the model's stored knowledge lives — the transformer's thinking engine

After attention mixes information between tokens, each token passes through a feed-forward network (FFN). This is a simple two-layer neural net applied independently to every position.

The FFN typically expands the representation (e.g., 4096 → 16384 dimensions), applies an activation function, then projects back down. It's where most of the model's parameters live — roughly 75% in many architectures.

Tiny explainer: why expand?
The expansion gives the network room to compute complex transformations. Think of it like having a bigger scratchpad — you can do more intermediate computation before compressing back down. Modern models use SwiGLU activation instead of simple ReLU for better quality.

Residual Stream & Layer Norm

What makes deep stacks trainable — the information highway that prevents gradient death

Each transformer block takes an input, processes it through attention and FFN, then adds the result back to the original input. This is called a residual connection or skip connection.

The "residual stream" is the running sum that flows through all layers. Each layer doesn't replace the signal — it adds its contribution on top. This means information can flow from input to output without being transformed at every step.

Tiny explainer: why residual connections?
Without them, each layer would have to learn an identity transformation just to pass information through. With 30+ layers stacked, the gradients would vanish or explode during backpropagation. Residual connections create a direct highway for gradients to flow.

Layer normalization stabilizes training by normalizing each token's representation (zero mean, unit variance) before it enters attention and FFN. This prevents any single dimension from dominating.

Next-Token Prediction

What the model actually outputs — softmax, sampling, and the generation loop

The final layer produces a score for every token in the vocabulary. These scores go through softmax to become probabilities that sum to 1. The model then picks one token — either greedily (highest probability) or by sampling.

Temperature controls how sharp or flat the distribution is:

  • temperature = 0: always pick the most likely token (deterministic)
  • temperature = 1.0: sample from the raw distribution
  • temperature > 1.0: flatten the distribution — more creative, more random

Top-k sampling limits choices to the k most likely tokens. Top-p (nucleus) sampling keeps only tokens whose cumulative probability reaches p.

Architecture vs Weights

What's broadly shared across modern LLMs, and what makes each one different

All modern LLMs share the same transformer-family skeleton. The differences come from:

  • Architecture choices: activation functions, positional encoding type, attention variants
  • Scale: number of parameters, context length, vocabulary size
  • Training data: what the model was trained on shapes its knowledge and behavior
  • Post-training: RLHF, DPO, constitutional AI — alignment techniques applied after pretraining

The architecture is like a car's chassis. The weights are the engine tuned by millions of miles of driving data.
