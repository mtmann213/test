Loop Engineering: From One-Shot AI to Feedback Systems

A practical academic-style course for understanding, practicing, and eventually employing reusable AI feedback loops in Hermes and adjacent workflows.

Accuracy note: “Loop engineering” is an emerging practitioner term, not a settled academic discipline. This course uses established ideas from cybernetics, control theory, software engineering, HCI prototyping, evaluation design, and organizational learning as conceptual scaffolding.
Day 1

Foundations: What Is Loop Engineering?

Learn to see AI workflows as feedback systems rather than isolated prompts.

Core thesis: loop engineering means designing systems that observe, reason, act, evaluate, update, and repeat. Evaluation is what turns repetition into directed improvement.
Goaltarget state
Contextinputs/tools
Actionagent work
Evaluationtests/rubric
Feedbackgap diagnosis
Updatenext policy

Academic lens

  • Cybernetics: studies control and communication in systems.
  • Control theory: closed-loop systems use observed error to correct behavior.
  • Scientific method: hypothesis → experiment → observation → revision.
  • Software engineering: code → test → fix → retest is a mature engineering loop.

Key distinction

One-shot AILoop-engineered AI
Answers onceActs, checks, improves
Prompt quality dominatesEvaluator + process dominate
Hard to compare across runsProduces comparable artifacts
Open-loop vs closed-loop: a one-shot prompt is usually open-loop; it acts without observing whether the result met the goal. A loop closes the circuit by checking output against a target and feeding that signal into the next action.

Day 2

Manual Loops Before Automation

Run a small loop by hand so you learn what is stable enough to automate.

Principle: automation should follow understanding. Manual loops reveal hidden inputs, weak schemas, unclear evaluation criteria, and places where human judgment matters.

Wizard-of-Oz framing

In HCI, a Wizard-of-Oz prototype lets a human simulate an automated system before engineers build it. For AI loops, manually run observe → act → evaluate → update before scheduling or agentifying it.

Day 3

Evaluation: The Steering Wheel

Design the signal that makes the loop improve in the right direction.

Useful analogy: the evaluator plays a role similar to a loss function in machine learning, but it is not literally training model weights. It steers the next action, artifact, prompt, rubric, or human decision.
Goodhart's Law: the popular formulation is “when a measure becomes a target, it ceases to be a good measure.” In loop engineering, pair scores with explanations, caveats, and weakest assumptions.

Rubric calculator

Score an idea from 1–5. This is a transparent teaching heuristic, not an objective truth machine.

Evaluator types

  • Deterministic: tests, schemas, link checks
  • Rubric: structured judgment
  • Comparative: A vs B
  • Human: strategic/taste approval
  • Model judge: useful for triage, not final truth

Rubric dimensions

  • Pain intensity
  • User clarity
  • Evidence strength
  • Feasibility
  • Distribution path
  • Testability

Check your understanding

The evaluator defines the feedback signal. Tests are best when possible; rubrics need explanations, uncertainty, and human review.
Day 4

One-Shot vs Loop Comparison

Use an ablation mindset: does the loop actually outperform a simpler prompt?

Loops are not always better. Use them when quality matters, the task repeats, evaluation is reliable, and the artifacts can compound.

Day 5

Codifying: From Process to Hermes Skill

A loop compounds when it becomes reusable procedural memory.

Skills prevent prompt drift. A successful process should become a reusable skill, checklist, rubric, template, or script so future agents can run it consistently.

Day 6

Automation: Building a Bounded System

Automate stable, low-risk, high-frequency parts first; keep strategic control human.

Appropriate autonomy: the goal is not maximum autonomy, but the right autonomy for the risk and maturity of the loop.

Automation ladder

Day 7

Reflection, Compounding, and Loop Portfolio

Move from one useful loop to a portfolio of compounding personal infrastructure.

Double-loop learning: in Argyris-and-Schön-style organizational learning, improvement can happen not just in actions, but in the governing rules. Here: improve outputs, then improve the workflow that creates outputs.

Single-loop

Improve the action: better summary, better patch, better ranking.

Double-loop

Improve the rule: better rubric, schema, skill, or test.

Triple-loop

Heuristic: improve loop selection itself — decide which loops deserve attention next.

Flashcards

Final deliverable: choose one loop to make real. Recommended: start with the Idea Opportunity Loop as a manually invoked Hermes skill, then graduate it to a weekly digest if it proves useful.

The real leverage comes from artifacts: skills, rubrics, docs, tests, and automation boundaries that improve future runs.
